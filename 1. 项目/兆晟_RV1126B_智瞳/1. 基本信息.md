  画框颜色逻辑改为：temp >= red → 红、temp >= yellow → 黄、temp >= green → 绿

  ---
  Step 2 — 增加逗留时间（dwell time）

  在 ZS_TEMP_ITEM_CFG 增加 iDwellTimeMs 字段，zs_vca_temp_alarm.c 内部维护每个跟踪目标的首次检测时间戳：

  目标出现 → 记录 firstSeenTime
  每帧判断 → (curTime - firstSeenTime) >= iDwellTimeMs → 才触发报警
  目标消失 → 清除记录

  ---
  Step 3 — 人形识别辅助温升

  在 ZS_VCA_TempAlarm_JudgeOne() 中增加逻辑：

  if (alarmMode == TEMP_RISE && 目标周围存在 VCA_OBJCLASS_HUMAN)
      effectiveThreshold = iAlarmThreshold + humanBonus(10~20°C)
  else
      effectiveThreshold = iAlarmThreshold

  ---
  Step 4 — 违禁品直接报警 + >100°C 强制报警

  if (category == PROHIBITED)
      → 立即报警（不检查温度）

  if (objectTemp >= 100.0f)
      → 强制报警（最高优先级，无论什么分类）

  ---
  Step 5 — 二次交叉验证

  温升报警和绝对温度报警同时成立才触发最终报警：

  if (alarmMode == TEMP_RISE)
      → (objectTemp - baseTemp >= threshold)  AND  (objectTemp >= absoluteMin)
      两个条件同时满足 → 才报警

  ---
  Step 6 — 学习功能接口

  预留回调函数指针，当发生报警/误报/漏报时回调给上层：

  typedef ZS_VOID (*ZS_TempAlarm_LearnCb)(ZS_TEMP_ALARM_RESULT *pResult, ZS_VOID *priv);

  zs_vca_temp_alarm.c — 判断流程

  ZS_VCA_TempAlarm_Judge()
    └─ JudgeOne() per object
         ├─ Priority 1: 违禁品 → 直接 RED
         ├─ Priority 2: >100°C → 强制 RED
         │
         ├─ 温升模式: judgeTemp = objTemp - baseTemp
         │     └─ 人形在旁? → judgeTemp -= 10°C
         ├─ 绝对模式: judgeTemp = objTemp
         │
         ├─ 三级阈值: >=red? RED : >=yellow? YELLOW : >=green? GREEN : NONE
         │
         ├─ 交叉验证: RED 且 bCrossValidate 时
         │     ├─ 温升模式 → 还需 absTemp >= iCrossAbsMin
         │     └─ 绝对模式 → 还需 rise >= iCrossRiseMin
         │
         ├─ 逗留延时: < iDwellTimeMs → NONE
         └─ 报警延时: < iAlarmDelayMs → NONE
