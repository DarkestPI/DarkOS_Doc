所有存储设备，包括固态硬盘、机械硬盘、移动硬盘、U盘、存储卡等等。都要先进行以下操作才能使用：

1. 按实际需求划分一个或多个分区。
2. 将所有分区格式化。

**分区**就是把存储空间划分为若干区域，方便资料存放和管理，例如电脑中的C盘、D盘等；而格式化就是在分区中建立指定的文件系统，以兼容不同的设备场景。

关于分区，我们后续会另行推文介绍，敬请期待。本文先给大家详细介绍一下分区格式化。

分区格式化最重要的参数就是文件系统，目前常见文件系统有FAT、NTFS、exFAT三种
## FAT 文件系统

FAT，分为 FAT（又叫FAT16）和 FAT23 两种；

这是目前兼容性最好的格式，所有电脑、平板、一体机、相机、手机、等等设备都支持这种格式。然而，其也有一些缺点：

| 文件系统       | 最小分区容量 | 最大分区容量 | 单文件大小上限 |
| ---------- | ------ | ------ | ------- |
| FAT（FAT16） | ——     | 2GB    | 2GB     |
| FAT32      | 512MB  | 2TB    | 4GB     |

## NTFS 文件系统

这是 **Windows操作系统** 应用最广泛的格式。在Windows操作系统中其有以下一些功能，是其它文件系统格式所不具备的：

NTFS 可以实现分区压缩、文件索引、磁盘配额、访问权限等等众多高级功能。其中访问权限甚至可以细化到单个文件或文件夹，功能是非常足弓强大的；

当然，这些高级功能一般仅限于在Windows操作系统中才能实现。在别的操作系统上，能识别分区并进行基本的读写操作就已经很好了。事实上，目前仍有很多其它操作系统和设备是不支持NTFS的。

另外，NTFS是“日志式”的文件系统，会记录详细的读写操作，因此不适合U盘、存储卡使用，因为这些设备的芯片是有读写次数的，NTFS频繁的读写操作会缩短其使用寿命。

最后，**NTFS分区容量和单文件大小理论上没有上限**。网上有些资料说的2TB，其实是在MBR分区格式下得出的数据，因为MBR最大只支持2TB容量，那么其上的NTFS也不能突破此限制而已。
##  exFAT 文件系统

顾名思义，这其实就是 FAT 的升级版本，消除了 FAT 的各种容量限制。但该格式是继 FAT 和 NTFS 之后才推出的，因此有不少设备不支持，兼容性稍差。


|           | FAT32              | NTFS                                 | exFAT                                |
| --------- | ------------------ | ------------------------------------ | ------------------------------------ |
| 操作系统      | Windows 95 OSR2 之后 | Windows NT3.0 之后                     | Windows CE 6/Vista SP1/ Windows 7 之后 |
| 最小簇       | 512 bytes          | 512 bytes                            | 512 bytes                            |
| 最大簇       | 64 KB              | 64 KB                                | 32768 KB                             |
| 同一目录最大文件数 | 65535              | 4294967295                           | 2796202                              |
| 最大单一文件    | 4 GB（2^32-1 bytes） | 256 TB（2^48-1 bytes）                 | 16 EB（2^64-1 bytes）                  |
| 最大卷容量     | 2 TB               | 2 TB~256 TB（受MBR影响，<br>GPT 分区格式可以更大） | 64ZB（理论值，目前支持到 128 PB）               |

# 文件系统的选择

1. 对于使用Windows操作系统，且固定安装在电脑主机内的固态硬盘、机械硬盘等，NTFS是不二之选，这没什么好说的了。

2. 对于大容量的移动硬盘，其内部盘体一般是机械硬盘或固态硬盘，可根据使用环境选NTFS或exFAT。

如主要是在Windows系统下使用，则首先NTFS。特别是Windows XP更是如此，因其默认不支持exFAT的，需打补丁才能用；

如要跨平台使用，例如除了Windows系统外，还要用于苹果Mac、Linux、安卓等等系统，则首先exFAT。

3. 对于U盘。建议2GB容量以下的用FAT；4GB容量以上的都用FAT32。可获得最好的兼容性，接入任何设备都能识别使用。

然而，如需存储大于4GB的大文件，则需使用exFAT了；但要有准备，这样的U盘在接入一些特别旧的设备上时可能会不能识别。

4. 对用于相机、摄像机、手机等设备的存储卡，一般都建议用FAT32，否则可能会无法识别。建议直接用设备的格式化功能更对存储卡处理，万无一失。


1.一般新买的移动硬盘、U盘、存储卡等出厂都已完成好分区和格式化，可直接使用；但这种出厂设置一般整个盘只分一个分区，而文件系统大都是FAT32，如不能满足使用需求，进行调整即可。

2.分区格式有两种：MBR和GPT（又称：GUID），可将其理解为整个设备全部可用存储空间的管理格式；

而常见文件系统有三种：FAT、NTFS、exFAT，可将其理解为设备上单个分区内部存储空间的管理格式。
